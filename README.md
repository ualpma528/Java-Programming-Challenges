## 🧰 Reto 01: Simulador de Combate RPG (29/04/2026)
El objetivo de este reto ha sido implementar la lógica de un sistema de combate por turnos usando POO.

### 🧠 Conceptos Aplicados
**Herencia:** Se ha definido una clase `Personaje` de la que heredan `Guerrero` y `Mago`.
**Polimorfismo:** El método `atacar()` se comporta de forma distinta según la especialización de la clase hija (usando `@Override`).
**Encapsulamiento:** Atributos protegidos y privados para asegurar la integridad de los datos (vida, fuerza, magia).
**Lógica de Control:** Implementación de bucles `while` y condicionales para gestionar los turnos de combate.
